project
경험한 프로젝트
프로젝트명 스마트팩토리 MES 웹 솔루션
담당 페이지 재고관리, 생산관리, 실적관리, 사원관리/등록
구현기능
  1. 입고관리에서 입고할 물품을 신청하거나 수정하고, 결재하는 시스템을 구현했습니다.
  2. 보유재고관리에서 보유중인 재고를 확인하고 묶음으로 처리된 보유재고를 클릭하면 상세 품목이 나오게끔 구현했습니다.
  3. 생산계획관리에서 생산계획을 수립하는 기능을 구현했습니다.
  4. 생산지시관리에서 수립한 생산계획을 지시할 수 있도록 구상하였습니다.
  5. 생산실적관리에서 생산 실적을 한눈에 볼 수 있도록 구상하였습니다.
  6. 사원등록/관리에서 사원을 등록하거나 수정하고 삭제할 수 있도록 구현하였습니다.
프로젝트 중
어려웠던 점
  • 부족한 MES관련 지식
    • ㄴ 아무래도 처음 접하는 MES이기에 관련 지식이 많이 없었고 프로그램을 구상하는데 시간을 상당히 소요하게 되었습니다.

  • 한정된 시간, 팀원 이탈
    • ㄴ 첫 프로젝트이기도 하고 알았다고 생각했던 것들이 실제로 적용하는 단계에서 뜻대로 동작하지 않는 경우가 많았고 오류가 많이 발생하여 해결하는데 많은 시간을 소요했습니다. 중간에 팀원이 이탈하는 일도 발생해 많은 어려움이 있었습니다.

  • 원하는 정보를 얻기 위한 데이터베이스 조작
    • ㄴ 브라우저에 표시되는 정보로 데이터베이스에 접근하여 값을 얻는데 중복되거나, 찾을 수 없다는 오류를 접했습니다.
적용한
해결방안 		1. 부족한 MES관련 지식
학원과 연계된 회사에 전화하여 관련 정보를 듣고 정리하며 유튜브나 기업소개에 올라온 자료를 사용해서 공부하였습니다.

2. 한정된 시간, 팀원 이탈
하고 싶었던건 많았지만 조금 타협해서 기존에 계획했던 프로젝트의 규모를 축소하여 발표하였습니다. 팀원의 이탈 문제는 남은 팀원끼리 합의해서 이탈한 팀원의 담당 파트를 나눠 가지는 방식으로 진행했습니다. 남은 팀원끼리 더 잘 해보자는 말도 하며 더 합심하는 계기로 삼았습니다.

3. 원하는 정보를 얻기 위한 데이터베이스 조작
input hidden으로 데이터의 sequence값을 숨긴 후 form안에 집어넣어 전송하는 방법으로 해결할 수 있었습니다.
느낀점 프로젝트 기획 단계에서 시간을 조금 더 쓰는 한이 있더라도 좀 더 정교하게 계획할 필요가 있음을 느꼈습니다. 특히 데이터베이스 설계는 중복되거나 없는 데이터 값을 많이 겪으면서 데이터 정규화를 통한 무결성이 중요하다는 점을 느꼈습니다.
